Schrittfolge fuer Installation/Start des MAPPERs

Installation

  • die Datei mapper.tgz in ein Verzeichnis Eurer Wahl entpacken z.B. /home/mh14/mak (das ist das Beispielverzeichnis, auf das ich mich im folgenden beziehen werde)
  • falls nicht vorhanden Java installieren, weil das Applet nichts speichern kann
  • die Icons sollten, wenn gewünscht, im selben Verzeichnis entpackt werden
  • für Linux (startMapper) und Windows (startMapper.bat) liegen Startdateien vor, in denen ggf. der Pfad der Java Installion angepasst werden sollte, oder die Pfadangabe rausnehmen falls das Java im PATH steht
  • damit der Mapper weiss, wo seine Karten liegen, solltet Ihr aus der Datei mapper.def die eine Zeile mit url=file://localhost/home/mh14/mak/maps in Eure eigene Konfigurationsdatei mapper.ini kopieren
  • die URL muss auf das Installationsverzeichnis (bzw. das Verzeichnis wo die Karten (Unterverzeichnis maps) liegen) angepasst werden.
  • um mit einer Karte für ein Mud neu zu beginnen, das Verzeichnis default im Verzeichnis maps unter einem anderen Namen kopieren (z.B. mg.mud.de)
  • Installation fertig

    Starthinweise

  • Aufruf:
  • Linux: startMapper demo_de
  • Windows: startMapper.bat demo_de
  • der Parameter demo_de entspricht dem Verzeichnis wo die Karten für das entsprechende Mud liegen (z.B. mg.mud.de)
  • der Mapper lauscht auf Port 2000 auf localhost
  • also werden alle Befehle, die man dorthin schickt wie die aus der Kommandozeile behandelt

    Bedienung

  • entweder über die Menüs, die Buttons oder die Kommandozeile
  • Hilfe im Mapper gibts mit "?" oder Menü
  • alternativ ist auf der Doku-Seite eine fast vollständige Dokumentation zu finden
  • die einfachsten Befehle fuer den Anfang:
    • ae n o so: Ausgaenge erzeugen
    • so: in Richtung bewegen
    • g 5: zu Knoten 5 springen (auch namen moeglich)
    • am Hochebene : neue Map mit angegebenen Namen erzeugen
    • sm Hochebene : zu angegebener Map umschalten
    • ai short Ein Raum mit Baum : Text zu Schluesselwort (z.B. short) fuer Raum speichern
    • zoom nodes 5 : so zoomen das maximal 5 Knoten sichtbar sind
    • Mausklick rechts oder doppelt : zu Punkt springen
    • Shift und Mausklick, bzw. aufziehen : Knoten markieren
    • der Mapper speichert nicht selber ab! also save benutzen
    • verlassen mit Q wie quit
    • .... in die Hilfe gucken da steht alles drin (dasselbe wie auf der Webseite)

      Nutzung mit TF

    • Wer meine tf Pakete nutzt, kann die Datei mapper.tf laden, ansonsten finden sich darin Beispiele wie den Mapper von tf aus nutzen kann.
    • die Pakete enthalten Unterstuetzung fuer den Mapper (mitlaufen, springen zu Knotenpunkten (dabei wichtig den Knoten mit ai ways knotenname denselben Namen wie im Wegesystem verpassen), mitmappen, speichern von infos (mapper.tf)
    • zum Initialisieren der Verbindung /init_mapper aufrufen, dabei wird eine World fuer den Mapper erzeugt und die Variable mapper auf 1 gesetzt, wenn man die Verbindung temporaer unterbrechen will, reicht es, die Variable auf 0 zu setzen.
    Wenn Ihr einen Fehler findet, oder Ideen für eine einfachere Benutzung oder Erweiterung des Mappers habt, schickt mir einfach eine E-Mail. Bei Fehlern wär ein Log der Ausgaben des Mappers sehr hilfreich.
